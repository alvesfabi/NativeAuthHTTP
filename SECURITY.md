# Security Policy

## Reporting Security Issues

**Please do not report security vulnerabilities through public GitHub issues.**

If you believe you have found a security vulnerability in this project, please report it to the Microsoft Security Response Center (MSRC) at:

**https://msrc.microsoft.com/create-report**

Alternatively, you can email **secure@microsoft.com**.

Please include the following information (as much as you can provide) to help us better understand the nature and scope of the possible issue:

* Type of issue (e.g. buffer overflow, SQL injection, cross-site scripting, etc.)
* Full paths of source file(s) related to the manifestation of the issue
* The location of the affected source code (tag/branch/commit or direct URL)
* Any special configuration required to reproduce the issue
* Step-by-step instructions to reproduce the issue
* Proof-of-concept or exploit code (if possible)
* Impact of the issue, including how an attacker might exploit the issue

This information will help us triage your report more quickly.

## Preferred Languages

We prefer all communications to be in English.

## Policy

Microsoft follows the principle of [Coordinated Vulnerability Disclosure](https://www.microsoft.com/en-us/msrc/cvd).

## More Information

For more information about Microsoft's security response process, please see:

* Microsoft Security Response Center: https://www.microsoft.com/msrc
* Microsoft Security TechCenter: https://technet.microsoft.com/security
* Microsoft Security Advisories: https://docs.microsoft.com/security-updates/

Thank you for helping keep Microsoft and our users safe!
